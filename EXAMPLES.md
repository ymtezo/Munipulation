# Munipulation 使用例 (Usage Examples)

このファイルは、Munipulationプラグインの使用例を示します。

## 例1: バックリンク付き移動 (Move with Backlink)

以下のような構造を作成してください:

```markdown
- ~~>[[]]
  - [[Project Ideas]] 新しいAI機能を追加する
  - [[Project Ideas]] ダークモード対応
  - [[Todo]] 週報を書く
```

指示行 (`~~>[[]]`) にカーソルを置いて、コマンドパレットから "Process outliner instructions" を実行すると:

- "新しいAI機能を追加する" と "ダークモード対応" が "Project Ideas" ノートに追加されます
- "週報を書く" が "Todo" ノートに追加されます
- 各アイテムには、元のノートへのバックリンクが自動的に追加されます

## 例2: Todoistタスク追加 (Add to Todoist)

まず、プラグイン設定でTodoist APIトークンを設定してください。

```markdown
- ~~>TODOIST
  - プロジェクトの進捗報告をする
  - ミーティングの議事録を作成する
  - コードレビューを完了する
```

指示行 (`~~>TODOIST`) にカーソルを置いて、コマンドパレットから "Process outliner instructions" を実行すると:

- 各サブアイテムがTodoistのタスクとして追加されます
- 成功すると通知が表示されます

## 例3: 複合的な使用 (Combined Usage)

異なる指示を同じノート内で使用できます:

```markdown
# 今日のタスク

- ~~>TODOIST
  - メールを返信する
  - 会議資料を準備する

# アイデアメモ

- ~~>[[]]
  - [[Future Features]] ノート間のリンク可視化
  - [[Research]] GraphQLの調査
```

各指示を個別に実行できます。

## ヒント (Tips)

1. **インデント**: タブまたは2-4スペースでサブアイテムをインデントしてください
2. **リストマーカー**: `-` または `*` を使用できます
3. **エラー処理**: リンクが見つからない場合、新しいノートが自動的に作成されます
4. **API制限**: Todoist APIには制限があるため、大量のタスクを一度に追加しないでください

## トラブルシューティング (Troubleshooting)

### "No instruction found on this line"

- 指示行にカーソルがあることを確認してください
- 指示が `~~>` で始まっていることを確認してください

### "No sub-items found to process"

- サブアイテムが正しくインデントされていることを確認してください
- 指示行の下に少なくとも1つのサブアイテムがあることを確認してください

### Todoistへの追加が失敗する

- 設定でAPIトークンが正しく設定されていることを確認してください
- インターネット接続を確認してください
